## 📄 `insertOne()` — Insert a Single Document

You already used `insertOne()` in [Section 2](2.InsertingDocuments.md). Here's the full syntax:

```javascript
db.<collection>.insertOne(document, options)
```

MongoDB documents can be as simple or as complex as you need:

```javascript
db.users.insertOne({
  name: "Gebin",
  age: 23,
  email: "gebin@example.com",
  skills: ["JavaScript", "MongoDB", "React"],
  contact: {
    phone: "9876543210",
    address: {
      city: "Kochi",
      state: "Kerala"
    }
  },
  education: [
    { degree: "BCA", year: 2022 },
    { degree: "MCA", year: 2024 }
  ]
})
```

> You can nest objects inside objects, store arrays of objects, mix data types — MongoDB is fully flexible with document structure.

## 📚 `insertMany()` — Insert Multiple Documents

When you need to insert more than one document at a time, pass an **array**:

```javascript
db.users.insertMany([
  { name: "George", age: 25, role: "Backend" },
  { name: "Anna",   age: 22, role: "Frontend" },
  { name: "Mark",   age: 30, role: "DevOps" }
])
```

**Output:**
```
{
  acknowledged: true,
  insertedIds: {
    '0': ObjectId('699d4ea03321373e437c290b'),
    '1': ObjectId('699d4ea03321373e437c290c'),
    '2': ObjectId('699d4ea03321373e437c290d')
  }
}
```

> **🧠 Why use `insertMany()` over multiple `insertOne()` calls?** It sends everything in a **single network request**, making it significantly faster for bulk inserts.

## 🆔 The `_id` Field

Every document in MongoDB has an `_id` field — a unique identifier. If you don't provide one, MongoDB generates an `ObjectId` automatically.

### Auto-generated

```javascript
db.users.insertOne({ name: "Test" })
// _id: ObjectId('65a1b2c3d4e5f6a7b8c9d0e1')  ← auto-generated
```

### Custom IDs

You can use your own `_id` — it can be a string, number, or anything:

```javascript
db.users.insertOne({ _id: "user_001", name: "Gebin", role: "Admin" })
```

```javascript
db.users.insertOne({ _id: 12345, name: "TestUser" })
```

> **⚠️** If you use custom IDs, it's your job to keep them unique. Duplicates will throw an error.

## 📦 Ordered vs Unordered Inserts

When using `insertMany()`, MongoDB inserts documents **in order** by default. If one fails, it **stops** and skips the rest.

### Ordered (default) — stops on error

```javascript
db.products.insertMany([
  { _id: 1, name: "Laptop" },
  { _id: 1, name: "Phone" },   // ❌ Duplicate! Stops here.
  { _id: 3, name: "Tablet" }   // ❌ Never reached
])
// Result: Only "Laptop" gets inserted
```

### Unordered — continues past errors

```javascript
db.products.insertMany(
  [
    { _id: 1, name: "Laptop" },  // ❌ Already exists
    { _id: 2, name: "Phone" },   // ✅ Inserted
    { _id: 3, name: "Tablet" }   // ✅ Inserted
  ],
  { ordered: false }
)
// Result: "Phone" and "Tablet" get inserted despite "Laptop" failing
```

> Use `{ ordered: false }` when you want to insert as much as possible and don't care about order — great for bulk imports.

## ❌ Common Errors

### Duplicate Key

```
MongoServerError: E11000 duplicate key error
```

**Fix:** Each `_id` must be unique. Let MongoDB auto-generate it, or make sure your custom IDs don't conflict.

### Document Too Large

```
MongoServerError: object to insert too large
```

**Fix:** Single documents can't exceed **16 MB**. For larger data, look into [GridFS](https://www.mongodb.com/docs/manual/core/gridfs/).

| What | How |
|:-----|:----|
| Insert one | `db.col.insertOne({ name: "A" })` |
| Insert many | `db.col.insertMany([{...}, {...}])` |
| Custom `_id` | `db.col.insertOne({ _id: "abc", ... })` |
| Unordered insert | `db.col.insertMany([...], { ordered: false })` |
| Max document size | 16 MB |
