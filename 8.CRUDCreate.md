## 📄 `insertOne()` — Insert a Single Document

`insertOne()` adds exactly **one document** to a collection. If the collection doesn't exist, MongoDB creates it automatically.

### Syntax

```javascript
db.<collection>.insertOne(document, options)
```

### Basic Example

```javascript
db.users.insertOne({
  name: "Gebin",
  age: 23,
  email: "gebin@example.com",
  role: "FullStackDev",
  skills: ["JavaScript", "MongoDB", "React"]
})
```

**Output:**
```
{
  acknowledged: true,
  insertedId: ObjectId('699d49a83321373e437c2907')
}
```

> **What the output tells us:**
> - `acknowledged: true` → MongoDB confirmed the write operation
> - `insertedId` → The unique `_id` automatically assigned to the document

### Inserting Nested Documents

MongoDB documents can contain other documents (objects) and arrays:

```javascript
db.users.insertOne({
  name: "Jerin",
  age: 23,
  contact: {
    email: "jerin@example.com",
    phone: "9876543210",
    address: {
      city: "Kochi",
      state: "Kerala",
      country: "India"
    }
  },
  skills: ["Python", "Django", "PostgreSQL"],
  education: [
    { degree: "BCA", year: 2022 },
    { degree: "MCA", year: 2024 }
  ]
})
```

**Output:**
```
{
  acknowledged: true,
  insertedId: ObjectId('699d4aaf3321373e437c2908')
}
```

> **💡 Key Takeaway:** MongoDB documents are incredibly flexible — you can nest objects inside objects, store arrays of objects, mix data types, and there's no fixed schema. This is one of the biggest advantages of a document database over relational databases.

## 📚 `insertMany()` — Insert Multiple Documents

When you need to insert more than one document at a time, use `insertMany()`. It takes an **array** of documents.

### Syntax

```javascript
db.<collection>.insertMany([document1, document2, ...], options)
```

### Basic Example

```javascript
db.users.insertMany([
  {
    name: "George",
    age: 25,
    role: "Backend",
    skills: ["Java", "Spring Boot"]
  },
  {
    name: "Anna",
    age: 22,
    role: "Frontend",
    skills: ["React", "CSS", "TypeScript"]
  },
  {
    name: "Mark",
    age: 30,
    role: "DevOps",
    skills: ["Docker", "Kubernetes", "AWS"]
  }
])
```

**Output:**
```
{
  acknowledged: true,
  insertedIds: {
    '0': ObjectId('699d4ea03321373e437c290b'),
    '1': ObjectId('699d4ea03321373e437c290c'),
    '2': ObjectId('699d4ea03321373e437c290d')
  }
}
```

> **Notice:** The output includes `insertedIds` (plural) — an object where each key is the index of the document in the array, and the value is the generated `_id`.

### Performance Benefit

`insertMany()` is significantly **faster** than calling `insertOne()` in a loop because:
- It sends all documents in a **single network request**
- MongoDB processes them as a **batch operation**
- Less round-trip overhead between your app and the database

> **🧠 Rule of thumb:** If you're inserting more than one document, **always** prefer `insertMany()` over multiple `insertOne()` calls.

## 🆔 The `_id` Field and ObjectId

Every document in MongoDB **must** have an `_id` field. It acts as the **primary key** — a unique identifier for that document within the collection.

### Auto-Generated IDs

If you don't provide an `_id`, MongoDB generates one for you as an `ObjectId`:

```javascript
db.users.insertOne({ name: "Test User" })
```

**Output:**
```
{
  acknowledged: true,
  insertedId: ObjectId('65a1b2c3d4e5f6a7b8c9d0e1')
}
```

An `ObjectId` is a **12-byte** value that encodes:

```
┌──────────┬──────────┬───────────┬──────────┐
│ Timestamp │ Machine  │ Process   │ Counter  │
│ (4 bytes) │ (3 bytes)│ (2 bytes) │ (3 bytes)│
└──────────┴──────────┴───────────┴──────────┘
```

- **Timestamp** → When the document was created (extractable!)
- **Machine** → Random value unique to the machine
- **Process** → Random value unique to the process
- **Counter** → Auto-incrementing counter

> **💡 Cool fact:** You can extract the creation timestamp from any ObjectId:
> ```javascript
> ObjectId('65a1b2c3d4e5f6a7b8c9d0e1').getTimestamp()
> // ISODate("2024-01-12T10:30:27.000Z")
> ```

### Custom IDs

You can provide your own `_id` if you prefer — it can be any type (string, number, etc.):

```javascript
db.users.insertOne({
  _id: "user_gebin_001",
  name: "Gebin",
  role: "Admin"
})
```

**Output:**
```
{
  acknowledged: true,
  insertedId: 'user_gebin_001'
}
```

```javascript
db.users.insertOne({
  _id: 12345,
  name: "TestUser",
  role: "Viewer"
})
```

> **⚠️ Warning:** If you use custom IDs, it's **your responsibility** to ensure they're unique. Inserting a document with a duplicate `_id` will throw an error.

## 📦 Ordered vs Unordered Inserts

When using `insertMany()`, MongoDB inserts documents **in order** by default. If one fails, it **stops** and doesn't insert the rest.

### Ordered Insert (Default)

```javascript
db.products.insertMany([
  { _id: 1, name: "Laptop",  price: 999 },
  { _id: 1, name: "Phone",   price: 699 },  // ❌ Duplicate _id!
  { _id: 3, name: "Tablet",  price: 499 }   // ❌ Never reached
])
```

**Output:**
```
MongoBulkWriteError: E11000 duplicate key error collection: GebinDB.products index: _id_
Result: {
  insertedCount: 1,   // Only the first document was inserted
  ...
}
```

> Only "Laptop" (index 0) gets inserted. "Phone" fails due to duplicate `_id`, and "Tablet" is **skipped** entirely.

### Unordered Insert

Pass `{ ordered: false }` to continue inserting even after failures:

```javascript
db.products.insertMany(
  [
    { _id: 1, name: "Laptop",  price: 999 },  // ❌ Fails (already exists)
    { _id: 2, name: "Phone",   price: 699 },  // ✅ Inserted
    { _id: 3, name: "Tablet",  price: 499 }   // ✅ Inserted
  ],
  { ordered: false }
)
```

**Output:**
```
MongoBulkWriteError: E11000 duplicate key error...
Result: {
  insertedCount: 2,   // Phone and Tablet were inserted despite Laptop failing
  ...
}
```

> **🧠 When to use unordered?**
> - When you want to insert as many documents as possible, regardless of individual failures
> - When the order of insertion doesn't matter
> - When importing bulk data where some duplicates might exist

## 🔒 Write Concerns

Write concerns let you control how MongoDB **acknowledges** write operations. This is about the trade-off between **speed** and **durability**.

### Default Write Concern (`w: 1`)

```javascript
db.users.insertOne(
  { name: "Gebin" },
  { writeConcern: { w: 1 } }
)
```

> Waits for the **primary** server to acknowledge the write. This is the default.

### No Acknowledgment (`w: 0`)

```javascript
db.users.insertOne(
  { name: "SpeedyUser" },
  { writeConcern: { w: 0 } }
)
```

> Fire-and-forget. The driver doesn't wait for any acknowledgment. **Fastest, but riskiest.**

### Majority Acknowledgment (`w: "majority"`)

```javascript
db.users.insertOne(
  { name: "SafeUser" },
  { writeConcern: { w: "majority" } }
)
```

> Waits until the write is confirmed by a **majority of replica set members**. **Safest, but slowest.**

| Write Concern | Speed | Durability | Use Case |
|:-------------|:-----:|:----------:|:---------|
| `w: 0` | ⚡ Fastest | ❌ None | Logging, metrics (data loss OK) |
| `w: 1` | ⚖️ Balanced | ✅ Primary | Most applications (default) |
| `w: "majority"` | 🐢 Slowest | ✅✅ Majority | Financial data, critical writes |

## ❌ Common Errors and How to Fix Them

### Duplicate Key Error

```
MongoServerError: E11000 duplicate key error collection: GebinDB.users index: _id_ dup key: { _id: "user_001" }
```

**Fix:** Each `_id` must be unique. Either let MongoDB auto-generate it, or ensure your custom IDs don't conflict.

### Document Validation Error

```
MongoServerError: Document failed validation
```

**Fix:** If the collection has a validator (schema), your document doesn't match the expected structure. Check the validation rules with:

```javascript
db.getCollectionInfos({ name: "users" })
```

### Exceeding Document Size Limit

```
MongoServerError: object to insert too large
```

**Fix:** A single document cannot exceed **16 MB**. If you need to store larger data, consider using [GridFS](https://www.mongodb.com/docs/manual/core/gridfs/).

## 📋 Summary Cheat Sheet

| Method | Description | Example |
|:-------|:------------|:--------|
| `insertOne(doc)` | Insert one document | `db.col.insertOne({ name: "A" })` |
| `insertMany([docs])` | Insert multiple documents | `db.col.insertMany([{...}, {...}])` |
| `{ ordered: false }` | Continue on error | `db.col.insertMany([...], { ordered: false })` |
| Custom `_id` | Use your own ID | `db.col.insertOne({ _id: "abc", ... })` |
| `writeConcern` | Control acknowledgment | `db.col.insertOne({...}, { writeConcern: { w: 1 } })` |
| Auto `ObjectId` | Default 12-byte unique ID | Automatically generated if `_id` omitted |
