# 4. First Stage Operator: Match

---

## What is `$match`?

`$match` filters documents тАФ it keeps only the ones that satisfy a condition and throws away the rest. It works exactly like the query you'd pass to `find()`, but inside a pipeline.

**Always place `$match` as early as possible** in your pipeline. The sooner you filter out documents you don't need, the less work the remaining stages have to do.

---

## Syntax

```javascript
{ $match: { <query> } }
```

The `<query>` is a standard MongoDB query тАФ the same one you'd use with `find()`. All the query operators you already know (`$gt`, `$lt`, `$in`, `$and`, `$or`, etc.) work here.

---

## Examples

### Simple match тАФ filter by a field value

```javascript
{ $match: { city: "New York" } }
```

_"Keep only documents where `city` is `New York`."_

### Match with comparison operator

```javascript
{ $match: { age: { $gt: 25 } } }
```

_"Keep only documents where `age` is greater than 25."_

### Match with `$and` тАФ multiple conditions

```javascript
{ $match: { $and: [ { gender: "female" }, { age: { $gt: 25 } } ] } }
```

_"Keep only documents where `gender` is `female` AND `age` is greater than 25."_

> **ЁЯТб Remember:** Passing multiple fields in a single object already acts as AND, so you can also write this as:
> ```javascript
> { $match: { gender: "female", age: { $gt: 25 } } }
> ```
> Both do the same thing. Use `$and` explicitly when you have multiple conditions on the **same field**.

---

## Using `$match` in a Pipeline

```javascript
// Get all active females from USA
db.persons.aggregate([
  { $match: { isActive: true, gender: "female", "company.location.country": "USA" } }
])
```

```javascript
// Count active users older than 30
db.persons.aggregate([
  { $match: { isActive: true, age: { $gt: 30 } } },
  { $count: "activeOver30" }
])
```

```javascript
// Average age of active male users, grouped by country
db.persons.aggregate([
  { $match: { isActive: true, gender: "male" } },
  { $group: { _id: "$company.location.country", avgAge: { $avg: "$age" } } },
  { $sort: { avgAge: -1 } }
])
```

---

## Note

> `$match` uses standard MongoDB queries and supports **all query operators** тАФ everything you learned in [CRUD тАФ Read](../7.CRUDRead.md) works inside `$match`.
