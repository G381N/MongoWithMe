# 3. Aggregation Expressions

---

## What are Expressions?

In aggregation pipelines, you often need to **refer to field values** from your documents. That's where expressions come in.

When you write `"$name"`, you're telling MongoDB: _"use the value of the `name` field from each document."_

---

## Why the `"$field"` Syntax?

You might wonder — why do we wrap it in quotes with a `$` in front?

- **Without quotes:** MongoDB would treat `$name` as a JavaScript variable — which doesn't exist, so it would break
- **With quotes `"$name"`:** MongoDB understands this as a **field reference** — it reads the value of the `name` field from the input document

```javascript
"$name"    // ✅ Refers to the "name" field of the document
$name      // ❌ JavaScript variable — not what we want
"name"     // ❌ Just the literal string "name", not the field value
```

> **Rule:** Whenever you want to use a field's value inside an aggregation stage, prefix it with `$` and wrap it in quotes.

---

## Example 1 — Simple Field Reference

```javascript
{ $group: { _id: "$age" } }
```

Here `"$age"` tells MongoDB: _"group the documents by the value of the `age` field."_

---

## Example 2 — Nested Field Reference

```javascript
{ $group: { _id: "$company.location.country" } }
```

This uses **dot notation** to reach into nested documents:

```
$company                          → first level property
  └── location                    → property of company
        └── country               → property of location
```

So `"$company.location.country"` means: _"go into `company`, then into `location`, then grab the value of `country`."_

Using our Persons data, this would group all persons by their country (USA, Italy, France, Germany, etc.).
