# 2. Stages & Stage Operators

---

## What are Stages?

A stage is a **single step** in the aggregation pipeline. Each stage takes in documents, does something to them, and passes the result to the next stage.

```
Document ‚Üí [Stage] ‚Üí Document ‚Üí [Stage] ‚Üí Document ‚Üí [Stage] ‚Üí Document
```

Every stage starts with a **stage operator** ‚Äî a keyword prefixed with `$`.

---

## Stage Operators

Below are the stage operators sorted from **most commonly used to least commonly used**. The ones marked with ‚≠ê are the ones you'll use the most.

---

### ‚≠ê `$match` ‚Äî Filter Documents

Filters documents based on a condition. Works exactly like `find()` but inside a pipeline.

**Always put `$match` as early as possible** ‚Äî it reduces the number of documents the rest of the pipeline has to process.

```javascript
// Get only active persons
db.persons.aggregate([
  { $match: { isActive: true } }
])
```

```javascript
// Active persons older than 30
db.persons.aggregate([
  { $match: { isActive: true, age: { $gt: 30 } } }
])
```

```javascript
// Persons from USA or Germany
db.persons.aggregate([
  { $match: { "company.location.country": { $in: ["USA", "Germany"] } } }
])
```

> You can use all the same query operators you already know ‚Äî `$gt`, `$lt`, `$in`, `$or`, etc.

---

### ‚≠ê `$group` ‚Äî Group & Calculate

Groups documents by a field and performs calculations (count, average, sum, etc.) on each group.

```javascript
// Count how many people per gender
db.persons.aggregate([
  {
    $group: {
      _id: "$gender",
      count: { $sum: 1 }
    }
  }
])
```

**Output:**
```
[
  { _id: 'female', count: 507 },
  { _id: 'male',   count: 493 }
]
```

The `_id` field is **what you're grouping by**. The `$` before a field name means _"use the value of this field"_.

**Accumulators** ‚Äî the calculations you can do inside `$group`:

| Accumulator | What It Does | Example |
|:------------|:-------------|:--------|
| `$sum` | Count or add up values | `{ $sum: 1 }` to count, `{ $sum: "$age" }` to total |
| `$avg` | Average | `{ $avg: "$age" }` |
| `$min` | Smallest value | `{ $min: "$age" }` |
| `$max` | Largest value | `{ $max: "$age" }` |
| `$push` | Collect all values into an array | `{ $push: "$name" }` |

```javascript
// Average age per eye color
db.persons.aggregate([
  {
    $group: {
      _id: "$eyeColor",
      avgAge: { $avg: "$age" }
    }
  }
])
```

```javascript
// Users per country with youngest and oldest
db.persons.aggregate([
  {
    $group: {
      _id: "$company.location.country",
      total: { $sum: 1 },
      youngest: { $min: "$age" },
      oldest: { $max: "$age" }
    }
  }
])
```

---

### ‚≠ê `$project` ‚Äî Reshape Documents

Controls which fields appear in the output. Include, exclude, rename, or create new fields.

```javascript
// Show only name and age, hide _id
db.persons.aggregate([
  {
    $project: {
      _id: 0,
      name: 1,
      age: 1
    }
  }
])
```

```javascript
// Pull nested country up to top level
db.persons.aggregate([
  {
    $project: {
      _id: 0,
      name: 1,
      country: "$company.location.country"
    }
  }
])
```

> `$project` with `1` = include, `0` = exclude. You can also assign a field to a new name like `country: "$company.location.country"`.

---

### ‚≠ê `$sort` ‚Äî Sort Results

Sorts documents by a field. `1` = ascending, `-1` = descending.

```javascript
// Sort by age, oldest first
db.persons.aggregate([
  { $sort: { age: -1 } }
])
```

```javascript
// Most popular fruits (combine with $group)
db.persons.aggregate([
  { $group: { _id: "$favoriteFruit", count: { $sum: 1 } } },
  { $sort: { count: -1 } }
])
```

---

### ‚≠ê `$limit` ‚Äî Limit Results

Returns only the first `n` documents.

```javascript
// Top 3 oldest people
db.persons.aggregate([
  { $sort: { age: -1 } },
  { $limit: 3 },
  { $project: { _id: 0, name: 1, age: 1 } }
])
```

---

### ‚≠ê `$count` ‚Äî Count Documents

Counts all documents at that point in the pipeline and outputs a single document with the total.

```javascript
// How many active users?
db.persons.aggregate([
  { $match: { isActive: true } },
  { $count: "activeUsers" }
])
```

**Output:**
```
[ { activeUsers: 485 } ]
```

---

### ‚≠ê `$skip` ‚Äî Skip Documents

Skips the first `n` documents. Useful for pagination when combined with `$limit`.

```javascript
// Skip first 5, get next 5 (page 2)
db.persons.aggregate([
  { $sort: { name: 1 } },
  { $skip: 5 },
  { $limit: 5 }
])
```

---

### ‚≠ê `$out` ‚Äî Write Results to a Collection

Writes the pipeline output into a **new collection** (or replaces an existing one). Must be the **last stage**.

```javascript
// Save all active users to a new collection
db.persons.aggregate([
  { $match: { isActive: true } },
  { $project: { _id: 0, name: 1, age: 1, gender: 1 } },
  { $out: "activePersons" }
])
```

After this, you'll have a new collection called `activePersons`:

```javascript
db.activePersons.find()
```

> **‚öÝÔ∏è** If the collection already exists, `$out` **replaces it entirely**.

---

### `$unwind` ‚Äî Flatten Arrays

Takes an array field and creates **one document per array element**. Useful when you want to group or count by array values.

```javascript
// Flatten the tags array
db.persons.aggregate([
  { $unwind: "$tags" },
  { $limit: 5 },
  { $project: { _id: 0, name: 1, tags: 1 } }
])
```

**Before:** One document with `tags: ["enim", "id", "velit"]`
**After:** Three separate documents, each with one tag.

```javascript
// Most common tags
db.persons.aggregate([
  { $unwind: "$tags" },
  { $group: { _id: "$tags", count: { $sum: 1 } } },
  { $sort: { count: -1 } },
  { $limit: 5 }
])
```

---

### `$addFields` / `$set` ‚Äî Add New Fields

Adds new fields to documents without removing existing ones. `$set` is just an alias for `$addFields`.

```javascript
db.persons.aggregate([
  {
    $addFields: {
      birthYear: { $subtract: [2025, "$age"] }
    }
  },
  { $limit: 3 },
  { $project: { _id: 0, name: 1, age: 1, birthYear: 1 } }
])
```

---

### `$unset` ‚Äî Remove Fields

Removes one or more fields from documents.

```javascript
db.persons.aggregate([
  { $unset: ["company", "tags", "index"] },
  { $limit: 3 }
])
```

---

### `$lookup` ‚Äî Join Collections

Pulls in documents from another collection ‚Äî the closest thing to a SQL JOIN.

```javascript
db.orders.aggregate([
  {
    $lookup: {
      from: "products",            // other collection
      localField: "productId",      // field in this collection
      foreignField: "_id",          // field in the other collection
      as: "productDetails"          // output array name
    }
  }
])
```

> We don't have a second collection in our dataset yet, but `$lookup` is important to know for real apps.

---

### `$merge` ‚Äî Merge Results into a Collection

Like `$out` but smarter ‚Äî it can **merge** into an existing collection instead of replacing it. Must be the last stage.

```javascript
db.persons.aggregate([
  { $match: { isActive: true } },
  { $merge: { into: "activePersons" } }
])
```

---

### `$replaceRoot` / `$replaceWith` ‚Äî Replace the Document

Replaces the entire document with a sub-document or computed value.

```javascript
// Make the company object the root document
db.persons.aggregate([
  { $replaceRoot: { newRoot: "$company" } },
  { $limit: 3 }
])
```

---

### `$bucket` / `$bucketAuto` ‚Äî Group into Ranges

Groups documents into **ranges** (buckets). Like `$group` but for numeric ranges.

```javascript
// Group people into age ranges
db.persons.aggregate([
  {
    $bucket: {
      groupBy: "$age",
      boundaries: [20, 25, 30, 35, 40, 45],
      default: "Other",
      output: { count: { $sum: 1 } }
    }
  }
])
```

`$bucketAuto` does the same but automatically picks the ranges for you:

```javascript
db.persons.aggregate([
  { $bucketAuto: { groupBy: "$age", buckets: 5 } }
])
```

---

### `$facet` ‚Äî Run Multiple Pipelines at Once

Runs **multiple aggregation pipelines** in parallel on the same data and returns all results in one document.

```javascript
db.persons.aggregate([
  {
    $facet: {
      "byGender": [
        { $group: { _id: "$gender", count: { $sum: 1 } } }
      ],
      "byFruit": [
        { $group: { _id: "$favoriteFruit", count: { $sum: 1 } } },
        { $sort: { count: -1 } }
      ]
    }
  }
])
```

> Returns one document with `byGender` and `byFruit` arrays ‚Äî both computed in a single query.

---

### `$sample` ‚Äî Random Documents

Returns a random selection of documents.

```javascript
// Get 3 random persons
db.persons.aggregate([
  { $sample: { size: 3 } }
])
```

---

### `$redact` ‚Äî Restrict Document Content

Controls access to document data at field level. Rarely used in basic apps.

---

### `$densify` ‚Äî Fill Gaps in Data

Adds missing values in a sequence (like filling missing dates in time-series data). Advanced use case.

---

### `$collStats` / `$indexStats` ‚Äî Collection & Index Info

Returns statistics about the collection or its indexes. For monitoring and debugging.

```javascript
db.persons.aggregate([
  { $collStats: { storageStats: {} } }
])
```

---

## Cheat Sheet

| Stage | What It Does | Must Know? |
|:------|:-------------|:----------:|
| `$match` | Filter documents | ‚≠ê Yes |
| `$group` | Group & calculate | ‚≠ê Yes |
| `$project` | Pick/rename/create fields | ‚≠ê Yes |
| `$sort` | Sort results | ‚≠ê Yes |
| `$limit` | Limit number of results | ‚≠ê Yes |
| `$count` | Count total documents | ‚≠ê Yes |
| `$skip` | Skip documents (pagination) | ‚≠ê Yes |
| `$out` | Save results to a collection | ‚≠ê Yes |
| `$unwind` | Flatten arrays | Yes |
| `$addFields` / `$set` | Add new computed fields | Yes |
| `$unset` | Remove fields | Yes |
| `$lookup` | Join with another collection | Yes |
| `$merge` | Merge into existing collection | Useful |
| `$replaceRoot` | Replace document with sub-doc | Useful |
| `$bucket` | Group into ranges | Useful |
| `$facet` | Run multiple pipelines at once | Useful |
| `$sample` | Get random documents | Nice to know |
| `$redact` | Restrict document content | Rare |
| `$densify` | Fill gaps in sequences | Rare |
| `$collStats` | Collection statistics | Debugging |
