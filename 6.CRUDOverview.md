# 🔄 6. CRUD Operations — Overview

> **CRUD** stands for **Create, Read, Update, Delete** — the four fundamental operations you'll perform on any database. In this section, we'll get a bird's-eye view of every CRUD method MongoDB gives us, along with their syntax and when to use each one.

## 💡 What is CRUD?

Every application that works with data performs four basic operations:

| Letter | Operation | What It Does |
|:------:|:---------:|:-------------|
| **C** | **Create** | Add new data to the database |
| **R** | **Read** | Retrieve existing data |
| **U** | **Update** | Modify existing data |
| **D** | **Delete** | Remove data from the database |

In MongoDB, these operations are performed on **documents** within **collections**. Think of a collection as a folder, and documents as the files inside it — except every "file" is a flexible JSON-like object.

---

## 🗺️ The Big Picture

Here's how MongoDB's methods map to each CRUD operation:

```
┌─────────────────────────────────────────────────────────────────┐
│                        CRUD in MongoDB                          │
├────────────────┬────────────────────────────────────────────────┤
│   CREATE (C)   │   insertOne()  ·  insertMany()                │
├────────────────┼────────────────────────────────────────────────┤
│   READ   (R)   │   find()  ·  findOne()                        │
├────────────────┼────────────────────────────────────────────────┤
│   UPDATE (U)   │   updateOne()  ·  updateMany()  · replaceOne()│
├────────────────┼────────────────────────────────────────────────┤
│   DELETE (D)   │   deleteOne()  ·  deleteMany()                │
└────────────────┴────────────────────────────────────────────────┘
```

All CRUD methods follow the pattern: `db.<collectionName>.<method>()`

## ➕ Create Operations

Create operations add new documents to a collection. If the collection doesn't exist, MongoDB creates it automatically.

### `insertOne(data, options)`

Inserts a **single document** into a collection.

```javascript
db.users.insertOne({
  name: "Gebin",
  age: 23,
  role: "FullStackDev"
})
```

**Output:**
```
{
  acknowledged: true,
  insertedId: ObjectId('699d49a83321373e437c2907')
}
```

### `insertMany(dataArray, options)`

Inserts **multiple documents** at once by passing an array.

```javascript
db.users.insertMany([
  { name: "Jerin", age: 23, role: "FullStackDev" },
  { name: "George", age: 25, role: "Backend" },
  { name: "Anna", age: 22, role: "Frontend" }
])
```

**Output:**
```
{
  acknowledged: true,
  insertedIds: {
    '0': ObjectId('699d4aaf3321373e437c2908'),
    '1': ObjectId('699d4ea03321373e437c290b'),
    '2': ObjectId('699d4ea03321373e437c290c')
  }
}
```

## 📖 Read Operations

Read operations retrieve documents from a collection. You can filter, sort, limit, and project (select specific fields).

### `find(filter, projection)`

Returns a **cursor** to all documents matching the filter.

```javascript
// Get ALL documents
db.users.find()

// Get documents where age is 23
db.users.find({ age: 23 })

// Get only name and role fields (projection)
db.users.find({}, { name: 1, role: 1 })
```

### `findOne(filter, projection)`

Returns the **first document** that matches the filter.

```javascript
db.users.findOne({ name: "Gebin" })
```

**Output:**
```
{
  _id: ObjectId('699d49a83321373e437c2907'),
  name: 'Gebin',
  age: 23,
  role: 'FullStackDev'
}
```

## ✏️ Update Operations

Update operations modify existing documents in a collection. MongoDB provides dedicated **update operators** like `$set`, `$unset`, `$inc`, etc.

### `updateOne(filter, update, options)`

Updates the **first document** that matches the filter.

```javascript
db.users.updateOne(
  { name: "Gebin" },
  { $set: { age: 24 } }
)
```

**Output:**
```
{
  acknowledged: true,
  insertedId: null,
  matchedCount: 1,
  modifiedCount: 1,
  upsertedCount: 0
}
```

### `updateMany(filter, update, options)`

Updates **all documents** that match the filter.

```javascript
db.users.updateMany(
  { role: "FullStackDev" },
  { $set: { verified: true } }
)
```

**Output:**
```
{
  acknowledged: true,
  insertedId: null,
  matchedCount: 2,
  modifiedCount: 2,
  upsertedCount: 0
}
```

### `replaceOne(filter, replacement, options)`

**Completely replaces** a document (except for the `_id` field). Unlike `updateOne()`, this doesn't use update operators — you provide the entire new document.

```javascript
db.users.replaceOne(
  { name: "George" },
  { name: "George", age: 26, role: "Senior Backend", verified: true }
)
```

> **⚠️ Warning:** `replaceOne()` will **overwrite** the entire document. Any fields not included in the replacement will be **lost**. Use `updateOne()` with `$set` if you only want to change specific fields.

---

## 🗑️ Delete Operations

Delete operations remove documents from a collection. Be careful — there's no "undo"!

### `deleteOne(filter, options)`

Deletes the **first document** that matches the filter.

```javascript
db.users.deleteOne({ name: "Anna" })
```

**Output:**
```
{
  acknowledged: true,
  deletedCount: 1
}
```

### `deleteMany(filter, options)`

Deletes **all documents** that match the filter.

```javascript
db.users.deleteMany({ verified: false })
```

**Output:**
```
{
  acknowledged: true,
  deletedCount: 3
}
```

> **⚠️ Danger Zone:** Running `db.users.deleteMany({})` with an **empty filter** will delete **every document** in the collection. Always double-check your filter!

## 📋 CRUD at a Glance

| Operation | Method | Target | Description |
|:---------:|:-------|:------:|:------------|
| **Create** | `insertOne(data)` | Single | Insert one document |
| **Create** | `insertMany([data])` | Multiple | Insert an array of documents |
| **Read** | `find(filter)` | Multiple | Find all matching documents |
| **Read** | `findOne(filter)` | Single | Find the first match |
| **Update** | `updateOne(filter, update)` | Single | Update the first match |
| **Update** | `updateMany(filter, update)` | Multiple | Update all matches |
| **Update** | `replaceOne(filter, doc)` | Single | Replace entire document |
| **Delete** | `deleteOne(filter)` | Single | Delete the first match |
| **Delete** | `deleteMany(filter)` | Multiple | Delete all matches |

> **🧠 Pattern:** Notice how every method comes in a `One` and `Many` variant? MongoDB keeps things consistent — once you learn one, you know the pattern for all of them.
