# 🔄 6. CRUD Operations — Overview

> **CRUD** stands for **Create, Read, Update, Delete** — the four fundamental operations for any database. This section gives you the big picture of every CRUD method in MongoDB.

## 💡 What is CRUD?

| Letter | Operation | What It Does |
|:------:|:---------:|:-------------|
| **C** | **Create** | Add new data |
| **R** | **Read** | Retrieve data |
| **U** | **Update** | Modify data |
| **D** | **Delete** | Remove data |

---

## 🗺️ The Big Picture

```
┌─────────────────────────────────────────────────────────────────┐
│                        CRUD in MongoDB                          │
├────────────────┬────────────────────────────────────────────────┤
│   CREATE (C)   │   insertOne()  ·  insertMany()                │
├────────────────┼────────────────────────────────────────────────┤
│   READ   (R)   │   find()  ·  findOne()                        │
├────────────────┼────────────────────────────────────────────────┤
│   UPDATE (U)   │   updateOne()  ·  updateMany()  · replaceOne()│
├────────────────┼────────────────────────────────────────────────┤
│   DELETE (D)   │   deleteOne()  ·  deleteMany()                │
└────────────────┴────────────────────────────────────────────────┘
```

All CRUD methods follow the same pattern: `db.<collectionName>.<method>()`

## ➕ Create

### `insertOne(data)` — Insert one document

```javascript
db.users.insertOne({
  name: "Gebin",
  age: 23,
  role: "FullStackDev"
})
```

### `insertMany([data])` — Insert multiple documents

```javascript
db.users.insertMany([
  { name: "Jerin", age: 23, role: "FullStackDev" },
  { name: "George", age: 25, role: "Backend" },
  { name: "Anna", age: 22, role: "Frontend" }
])
```

> **📖 Deep Dive:** [CRUD — Create (Deep Dive)](8.CRUDCreate.md) covers `insertMany()`, ordered/unordered inserts, custom IDs, and more.

## 📖 Read

### `find(filter)` — Find all matching documents

```javascript
db.users.find()                          // all documents
db.users.find({ age: 23 })              // filter by field
db.users.find({}, { name: 1, role: 1 }) // with projection
```

### `findOne(filter)` — Find the first match

```javascript
db.users.findOne({ name: "Gebin" })
```

> **📖 Deep Dive:** [CRUD — Read (Deep Dive)](7.CRUDRead.md) covers comparison operators, sorting, limiting, projection, and more.

## ✏️ Update

### `updateOne(filter, update)` — Update the first match

```javascript
db.users.updateOne(
  { name: "Gebin" },
  { $set: { age: 24 } }
)
```

### `updateMany(filter, update)` — Update all matches

```javascript
db.users.updateMany(
  { role: "FullStackDev" },
  { $set: { verified: true } }
)
```

### `replaceOne(filter, newDocument)` — Replace an entire document

```javascript
db.users.replaceOne(
  { name: "George" },
  { name: "George", age: 26, role: "Senior Backend", verified: true }
)
```

> **⚠️** `replaceOne()` **overwrites** the entire document. Fields not included are **lost**. Use `updateOne()` with `$set` if you only want to change specific fields.

---

## 🗑️ Delete

### `deleteOne(filter)` — Delete the first match

```javascript
db.users.deleteOne({ name: "Anna" })
```

### `deleteMany(filter)` — Delete all matches

```javascript
db.users.deleteMany({ verified: false })
```

> **⚠️** `deleteMany({})` with an empty filter deletes **everything** in the collection. Be careful!

---

## 📋 CRUD at a Glance

| Operation | Method | Description |
|:---------:|:-------|:------------|
| **Create** | `insertOne(data)` | Insert one document |
| **Create** | `insertMany([data])` | Insert multiple documents |
| **Read** | `find(filter)` | Find all matching documents |
| **Read** | `findOne(filter)` | Find the first match |
| **Update** | `updateOne(filter, update)` | Update the first match |
| **Update** | `updateMany(filter, update)` | Update all matches |
| **Update** | `replaceOne(filter, doc)` | Replace entire document |
| **Delete** | `deleteOne(filter)` | Delete the first match |
| **Delete** | `deleteMany(filter)` | Delete all matches |

> **🧠 Notice the pattern:** Every method comes in a `One` and `Many` variant. Once you learn one, you know them all.
