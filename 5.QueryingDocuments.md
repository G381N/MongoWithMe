## 🎯 Basic Filtering

So far we've used `find()` with no arguments to get all documents. But the real power comes when you pass a **query object** — a document that describes what you're looking for.

### Filtering by String

Find all documents where `name` is `"Gebin"`:

```javascript
db.mydata.find({ name: "Gebin" })
```

**Output:**
```
[
  {
    _id: ObjectId('699d49a83321373e437c2907'),
    name: 'Gebin',
    age: 23,
    phone: '9876543210',
    role: 'FullStackDev',
    socials: { hasGithub: true, hasLinkedin: true },
    education: [ 'BCA', 'MCA' ]
  }
]
```

> **How it works:** The query object `{ name: "Gebin" }` tells MongoDB _"give me every document where the `name` field equals `Gebin`."_

### Filtering by Number

You can filter by any field type — numbers work just as well:

```javascript
db.mydata.find({ age: 25 })
```

**Output:**
```
[
  {
    _id: ObjectId('699d4ea03321373e437c290b'),
    name: 'George',
    age: 25,
    phone: '9876543210',
    role: 'unavailable',
    socials: { hasGithub: true, hasLinkedin: true },
    education: [ 'BCA', 'MCA' ]
  }
]
```

### Multiple Filters (AND)

Pass multiple fields in the query object to combine conditions — MongoDB treats them as **AND**:

```javascript
db.mydata.find({ age: 23, role: "FullStackDev" })
```

This returns documents where `age` is 23 **AND** `role` is "FullStackDev".

## 🔢 Counting Results

Sometimes you don't need the actual documents — you just want to know **how many** match your query. Append `.count()` to the cursor:

```javascript
db.mydata.find({ age: 25 }).count()
```

**Output:**
```
1
```

Another example — how many people are 23 years old?

```javascript
db.mydata.find({ age: 23 }).count()
```

**Output:**
```
2
```

> **💡 Tip:** You can also count all documents in a collection:
> ```javascript
> db.mydata.find().count()
> ```
> **Output:**
> ```
> 5
> ```

## 🏠 Querying Nested Documents

Remember the `socials` field we added in the [previous section](4.UpdatingDocuments.md)? It's a **nested document** (an object inside another object). To query fields inside nested documents, use **dot notation**.

### Dot Notation

Use a dot (`.`) to "reach into" nested fields, and **wrap the key in quotes**:

```javascript
db.mydata.find({ "socials.hasGithub": true }).count()
```

**Output:**
```
5
```

This reads: _"Find all documents where the `hasGithub` field inside the `socials` object is `true`."_

### Another Example

Let's find users who have LinkedIn:

```javascript
db.mydata.find({ "socials.hasLinkedin": true })
```

### ⚠️ Important: Quotes Are Required!

You **must** wrap dot-notation keys in quotes. Without quotes, JavaScript interprets the dot as a property access and the shell will throw a syntax error:

```javascript
// ✅ Correct — key in quotes
db.mydata.find({ "socials.hasGithub": true })

// ❌ Wrong — will cause a syntax error
db.mydata.find({ socials.hasGithub: true })
```

---

## 📋 Quick Reference

| Command | What It Does |
|:--------|:-------------|
| `db.<col>.find({ field: value })` | Find documents where field equals value |
| `db.<col>.find({ a: 1, b: 2 })` | Find where field `a` is 1 AND `b` is 2 |
| `db.<col>.find({...}).count()` | Count matching documents |
| `db.<col>.find({ "nested.field": value })` | Query inside nested documents |
