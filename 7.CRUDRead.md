## 🎯 Filtering with Query Objects

You already know basic filtering from [Section 5](5.QueryingDocuments.md). Here's a quick recap:

```javascript
// Exact match
db.mydata.find({ role: "FullStackDev" })

// Multiple conditions (AND)
db.mydata.find({ age: 23, role: "FullStackDev" })
```

---

## ⚖️ Comparison Operators

When you need more than exact matches, use comparison operators. They all start with `$`.

### The Operators

| Operator | Meaning | Example |
|:--------:|:--------|:--------|
| `$gt` | Greater than | `{ age: { $gt: 23 } }` |
| `$gte` | Greater than or equal | `{ age: { $gte: 23 } }` |
| `$lt` | Less than | `{ age: { $lt: 25 } }` |
| `$lte` | Less than or equal | `{ age: { $lte: 23 } }` |
| `$eq` | Equal (same as exact match) | `{ age: { $eq: 25 } }` |
| `$ne` | Not equal | `{ age: { $ne: 25 } }` |
| `$in` | Matches any value in array | `{ role: { $in: ["Frontend", "Backend"] } }` |
| `$nin` | Does not match any | `{ role: { $nin: ["DevOps"] } }` |

### Examples

Find everyone older than 23:

```javascript
db.mydata.find({ age: { $gt: 23 } })
```

**Output:**
```
[
  { ..., name: 'George', age: 25, ... },
  { ..., name: 'Mark',   age: 30, ... }
]
```

### Range Queries

Combine operators on the same field for ranges:

```javascript
db.mydata.find({ age: { $gt: 22, $lt: 26 } })
```

This gives you everyone between 22 and 26 (exclusive):

**Output:**
```
[
  { ..., name: 'Gebin',  age: 23, ... },
  { ..., name: 'Jerin',  age: 23, ... },
  { ..., name: 'George', age: 25, ... }
]
```

## 🧩 Logical Operators

### `$or` — Match any condition

```javascript
db.mydata.find({
  $or: [
    { role: "Frontend" },
    { role: "Backend" }
  ]
})
```

### `$and` — Match all conditions (explicit)

```javascript
db.mydata.find({
  $and: [
    { age: { $gte: 23 } },
    { role: "FullStackDev" }
  ]
})
```

> **💡 Tip:** You rarely need `$and` since passing multiple fields already acts as AND. Use `$and` when you have multiple conditions on the **same field**.

## 🔢 Counting Results

```javascript
db.mydata.find({ age: 23 }).count()
```

```
2
```

Or use the newer `countDocuments()`:

```javascript
db.mydata.countDocuments({ role: "FullStackDev" })
```

```
2
```

## 🔁 Iterating with `.forEach()`

Process each document individually:

```javascript
db.mydata.find().forEach((doc) => {
  print(`Name: ${doc.name}, Age: ${doc.age}`)
})
```

**Output:**
```
Name: Gebin, Age: 23
Name: Jerin, Age: 23
Name: George, Age: 25
Name: Anna, Age: 22
Name: Mark, Age: 30
```

## 🎭 Projection — Choosing Fields

By default, `find()` returns all fields. Use **projection** (second argument) to pick which fields to show.

### Include specific fields

```javascript
db.mydata.find({}, { name: 1, role: 1 })
```

**Output:**
```
[
  { _id: ObjectId('...'), name: 'Gebin',  role: 'FullStackDev' },
  { _id: ObjectId('...'), name: 'Jerin',  role: 'FullStackDev' },
  { _id: ObjectId('...'), name: 'George', role: 'Backend' },
  ...
]
```

> `_id` is always included by default. To hide it, add `_id: 0`.

### Exclude specific fields

```javascript
db.mydata.find({}, { socials: 0, _id: 0 })
```

> **⚠️ Rule:** You can't mix inclusions and exclusions (except `_id`). Either include fields or exclude fields — not both.

## 📊 Sorting Results

```javascript
// Ascending (1)
db.mydata.find().sort({ age: 1 })

// Descending (-1)
db.mydata.find().sort({ age: -1 })
```

**Output (descending):**
```
[
  { ..., name: 'Mark',   age: 30, ... },
  { ..., name: 'George', age: 25, ... },
  { ..., name: 'Gebin',  age: 23, ... },
  { ..., name: 'Jerin',  age: 23, ... },
  { ..., name: 'Anna',   age: 22, ... }
]
```

---

## 📏 Limiting & Skipping

### Limit results

```javascript
db.mydata.find().limit(3)
```

### Skip + Limit (pagination)

```javascript
db.mydata.find().skip(2).limit(2)
```

This skips the first 2 results and returns the next 2 — useful for building pagination.

## 🔗 Chaining It All Together

The real power comes from combining everything in one query:

```javascript
db.mydata
  .find({ age: { $gte: 22 } }, { name: 1, age: 1, role: 1, _id: 0 })
  .sort({ age: -1 })
  .skip(1)
  .limit(3)
```

_"Find users aged 22+, show only name/age/role, sort by age descending, skip the first result, return 3."_

**Output:**
```
[
  { name: 'George', age: 25, role: 'Backend' },
  { name: 'Gebin',  age: 23, role: 'FullStackDev' },
  { name: 'Jerin',  age: 23, role: 'FullStackDev' }
]
```

## 📋 Cheat Sheet

| What | How | Example |
|:-----|:----|:--------|
| All docs | `find()` | `db.col.find()` |
| Filter | `find({filter})` | `db.col.find({ age: 23 })` |
| First match | `findOne({filter})` | `db.col.findOne({ name: "A" })` |
| Greater than | `$gt` | `{ age: { $gt: 25 } }` |
| Less than | `$lt` | `{ age: { $lt: 25 } }` |
| Not equal | `$ne` | `{ age: { $ne: 23 } }` |
| In list | `$in` | `{ role: { $in: ["A","B"] } }` |
| OR | `$or` | `{ $or: [{...}, {...}] }` |
| Count | `.count()` | `db.col.find({}).count()` |
| Include fields | `{ field: 1 }` | `db.col.find({}, { name: 1 })` |
| Exclude fields | `{ field: 0 }` | `db.col.find({}, { age: 0 })` |
| Sort asc | `.sort({ f: 1 })` | `db.col.find().sort({ age: 1 })` |
| Sort desc | `.sort({ f: -1 })` | `db.col.find().sort({ age: -1 })` |
| Limit | `.limit(n)` | `db.col.find().limit(5)` |
| Skip | `.skip(n)` | `db.col.find().skip(10)` |
| Nested query | `"a.b"` | `db.col.find({ "a.b": true })` |
