## 🧪 Setup — Sample Data

To follow along, let's make sure we have some sample data. Run this in your `mongosh` shell:

```javascript
use GebinDB
```

```javascript
db.mydata.insertMany([
  { name: "Gebin",  age: 23, role: "FullStackDev", socials: { hasGithub: true, hasLinkedin: true } },
  { name: "Jerin",  age: 23, role: "FullStackDev", socials: { hasGithub: true, hasLinkedin: false } },
  { name: "George", age: 25, role: "Backend",      socials: { hasGithub: true, hasLinkedin: true } },
  { name: "Anna",   age: 22, role: "Frontend",     socials: { hasGithub: false, hasLinkedin: true } },
  { name: "Mark",   age: 30, role: "DevOps",       socials: { hasGithub: true, hasLinkedin: true } }
])
```

Now we're ready to query!

## 🔎 `find()` vs `findOne()`

These are your two primary read methods. Let's understand the difference:

### `find()` — Returns Multiple Documents

`find()` returns a **cursor** — an iterable object pointing to all documents matching your query.

```javascript
db.mydata.find()
```

**Output:**
```
[
  { _id: ObjectId('...'), name: 'Gebin',  age: 23, role: 'FullStackDev', ... },
  { _id: ObjectId('...'), name: 'Jerin',  age: 23, role: 'FullStackDev', ... },
  { _id: ObjectId('...'), name: 'George', age: 25, role: 'Backend',      ... },
  { _id: ObjectId('...'), name: 'Anna',   age: 22, role: 'Frontend',     ... },
  { _id: ObjectId('...'), name: 'Mark',   age: 30, role: 'DevOps',       ... }
]
```

> **💡 Tip:** When there are many documents, `mongosh` will show the first 20 and prompt you to type `it` (iterate) to see more.

### `findOne()` — Returns a Single Document

`findOne()` returns the **first document** that matches the filter — not a cursor, but the actual document object.

```javascript
db.mydata.findOne({ name: "Gebin" })
```

**Output:**
```
{
  _id: ObjectId('699d49a83321373e437c2907'),
  name: 'Gebin',
  age: 23,
  role: 'FullStackDev',
  socials: { hasGithub: true, hasLinkedin: true }
}
```

> **🧠 When to use which?**
> | Scenario | Use |
> |:---------|:----|
> | Need all matching documents | `find()` |
> | Need just one document (e.g., lookup by ID) | `findOne()` |
> | Checking if a document exists | `findOne()` (returns `null` if not found) |

## 🎯 Filtering with Query Objects

The first argument to `find()` and `findOne()` is a **query object** — a document that specifies the conditions to match.

### Exact Match

```javascript
db.mydata.find({ role: "FullStackDev" })
```

**Output:**
```
[
  { _id: ObjectId('...'), name: 'Gebin', age: 23, role: 'FullStackDev', ... },
  { _id: ObjectId('...'), name: 'Jerin', age: 23, role: 'FullStackDev', ... }
]
```

### Multiple Conditions (AND)

Pass multiple fields in the query object — MongoDB treats them as an **AND** condition:

```javascript
db.mydata.find({ age: 23, role: "FullStackDev" })
```

This returns documents where **both** `age` is 23 **AND** `role` is "FullStackDev".

## ⚖️ Comparison Operators

Sometimes you need more than exact matches. MongoDB provides comparison operators that let you query ranges of values. All comparison operators start with `$`.

### `$gt` — Greater Than

Find documents where `age` is **greater than** 23:

```javascript
db.mydata.find({ age: { $gt: 23 } })
```

**Output:**
```
[
  { _id: ObjectId('...'), name: 'George', age: 25, ... },
  { _id: ObjectId('...'), name: 'Mark',   age: 30, ... }
]
```

### `$gte` — Greater Than or Equal To

Find documents where `age` is **greater than or equal to** 23:

```javascript
db.mydata.find({ age: { $gte: 23 } })
```

**Output:**
```
[
  { _id: ObjectId('...'), name: 'Gebin',  age: 23, ... },
  { _id: ObjectId('...'), name: 'Jerin',  age: 23, ... },
  { _id: ObjectId('...'), name: 'George', age: 25, ... },
  { _id: ObjectId('...'), name: 'Mark',   age: 30, ... }
]
```

### `$lt` — Less Than

Find documents where `age` is **less than** 25:

```javascript
db.mydata.find({ age: { $lt: 25 } })
```

**Output:**
```
[
  { _id: ObjectId('...'), name: 'Gebin', age: 23, ... },
  { _id: ObjectId('...'), name: 'Jerin', age: 23, ... },
  { _id: ObjectId('...'), name: 'Anna',  age: 22, ... }
]
```

### `$lte` — Less Than or Equal To

Find documents where `age` is **less than or equal to** 23:

```javascript
db.mydata.find({ age: { $lte: 23 } })
```

**Output:**
```
[
  { _id: ObjectId('...'), name: 'Gebin', age: 23, ... },
  { _id: ObjectId('...'), name: 'Jerin', age: 23, ... },
  { _id: ObjectId('...'), name: 'Anna',  age: 22, ... }
]
```

### Combining Operators (Range Queries)

You can combine multiple comparison operators on the same field to create **range queries**:

```javascript
db.mydata.find({ age: { $gt: 22, $lt: 26 } })
```

This reads: _"Find all documents where age is greater than 22 AND less than 26"_

**Output:**
```
[
  { _id: ObjectId('...'), name: 'Gebin',  age: 23, ... },
  { _id: ObjectId('...'), name: 'Jerin',  age: 23, ... },
  { _id: ObjectId('...'), name: 'George', age: 25, ... }
]
```

> **📋 Quick Reference:**
> | Operator | Meaning | Example |
> |:--------:|:--------|:--------|
> | `$gt` | Greater than | `{ age: { $gt: 25 } }` |
> | `$gte` | Greater than or equal | `{ age: { $gte: 25 } }` |
> | `$lt` | Less than | `{ age: { $lt: 25 } }` |
> | `$lte` | Less than or equal | `{ age: { $lte: 25 } }` |
> | `$eq` | Equal (same as exact match) | `{ age: { $eq: 25 } }` |
> | `$ne` | Not equal | `{ age: { $ne: 25 } }` |
> | `$in` | Matches any value in array | `{ role: { $in: ["Frontend", "Backend"] } }` |
> | `$nin` | Does not match any value | `{ role: { $nin: ["DevOps"] } }` |

## 🧩 Logical Operators

Need more complex conditions? Use logical operators:

### `$or` — Match Any Condition

```javascript
db.mydata.find({
  $or: [
    { role: "Frontend" },
    { role: "Backend" }
  ]
})
```

**Output:**
```
[
  { _id: ObjectId('...'), name: 'George', age: 25, role: 'Backend',  ... },
  { _id: ObjectId('...'), name: 'Anna',   age: 22, role: 'Frontend', ... }
]
```

### `$and` — Match All Conditions (Explicit)

```javascript
db.mydata.find({
  $and: [
    { age: { $gte: 23 } },
    { role: "FullStackDev" }
  ]
})
```

> **💡 Tip:** You rarely need `$and` explicitly since passing multiple fields to a query object already acts as AND. But `$and` is useful when you have multiple conditions on the **same field**.

---

## 🔢 Counting Results

Use `.count()` or `.countDocuments()` to count how many documents match a query:

```javascript
db.mydata.find({ age: 23 }).count()
```

**Output:**
```
2
```

You can also use `countDocuments()` directly on the collection:

```javascript
db.mydata.countDocuments({ role: "FullStackDev" })
```

**Output:**
```
2
```

> **💡 Note:** `.count()` is a convenience method on the cursor. `countDocuments()` is the official recommended method in newer versions of MongoDB.

## 🔁 Iterating with `.forEach()`

The cursor returned by `find()` has a `.forEach()` method that lets you process each document:

```javascript
db.mydata.find().forEach((doc) => {
  printjson(doc)
})
```

This prints each document in pretty JSON format. You can also do custom processing:

```javascript
db.mydata.find().forEach((doc) => {
  print(`Name: ${doc.name}, Age: ${doc.age}`)
})
```

**Output:**
```
Name: Gebin, Age: 23
Name: Jerin, Age: 23
Name: George, Age: 25
Name: Anna, Age: 22
Name: Mark, Age: 30
```

## 🎭 Projection — Selecting Specific Fields

By default, `find()` returns **all fields** in a document. Use **projection** (the second argument) to specify which fields to include or exclude.

### Include Specific Fields

```javascript
db.mydata.find({}, { name: 1, role: 1 })
```

**Output:**
```
[
  { _id: ObjectId('...'), name: 'Gebin',  role: 'FullStackDev' },
  { _id: ObjectId('...'), name: 'Jerin',  role: 'FullStackDev' },
  { _id: ObjectId('...'), name: 'George', role: 'Backend' },
  { _id: ObjectId('...'), name: 'Anna',   role: 'Frontend' },
  { _id: ObjectId('...'), name: 'Mark',   role: 'DevOps' }
]
```

> **Note:** `_id` is **always included** by default. To exclude it, set `_id: 0`.

### Exclude Specific Fields

```javascript
db.mydata.find({}, { socials: 0, _id: 0 })
```

This returns all fields **except** `socials` and `_id`.

> **⚠️ Rule:** You **cannot mix** inclusions and exclusions in a projection (except for `_id`). Either include specific fields or exclude specific fields — not both.

## 📊 Sorting Results

Use `.sort()` to order your results:

```javascript
// Sort by age ascending (1 = ascending)
db.mydata.find().sort({ age: 1 })
```

**Output:**
```
[
  { ..., name: 'Anna',   age: 22, ... },
  { ..., name: 'Gebin',  age: 23, ... },
  { ..., name: 'Jerin',  age: 23, ... },
  { ..., name: 'George', age: 25, ... },
  { ..., name: 'Mark',   age: 30, ... }
]
```

```javascript
// Sort by age descending (-1 = descending)
db.mydata.find().sort({ age: -1 })
```

**Output:**
```
[
  { ..., name: 'Mark',   age: 30, ... },
  { ..., name: 'George', age: 25, ... },
  { ..., name: 'Gebin',  age: 23, ... },
  { ..., name: 'Jerin',  age: 23, ... },
  { ..., name: 'Anna',   age: 22, ... }
]
```

---

## 📏 Limiting Results

Use `.limit()` to cap the number of results returned:

```javascript
db.mydata.find().limit(3)
```

**Output:**
```
[
  { ..., name: 'Gebin',  age: 23, ... },
  { ..., name: 'Jerin',  age: 23, ... },
  { ..., name: 'George', age: 25, ... }
]
```

Combine with `.skip()` for **pagination**:

```javascript
// Skip first 2 results, then return next 2
db.mydata.find().skip(2).limit(2)
```

**Output:**
```
[
  { ..., name: 'George', age: 25, ... },
  { ..., name: 'Anna',   age: 22, ... }
]
```

## 🔗 Chaining It All Together

One of the most powerful features of MongoDB's cursor is **method chaining**. You can combine `find`, `sort`, `skip`, `limit`, and `projection` into a single elegant query:

```javascript
db.mydata
  .find({ age: { $gte: 22 } }, { name: 1, age: 1, role: 1, _id: 0 })
  .sort({ age: -1 })
  .skip(1)
  .limit(3)
```

This reads: _"Find users aged 22+, show only name/age/role, sort by age descending, skip the first result, and return 3."_

**Output:**
```
[
  { name: 'George', age: 25, role: 'Backend' },
  { name: 'Gebin',  age: 23, role: 'FullStackDev' },
  { name: 'Jerin',  age: 23, role: 'FullStackDev' }
]
```

## 📋 Summary Cheat Sheet

| Method / Operator | Description | Example |
|:------------------|:------------|:--------|
| `find()` | Get all matching docs | `db.col.find({ age: 23 })` |
| `findOne()` | Get first matching doc | `db.col.findOne({ name: "Gebin" })` |
| `{ field: value }` | Exact match filter | `{ role: "Backend" }` |
| `$gt`, `$gte` | Greater than (or equal) | `{ age: { $gt: 25 } }` |
| `$lt`, `$lte` | Less than (or equal) | `{ age: { $lt: 25 } }` |
| `$eq`, `$ne` | Equal / Not equal | `{ age: { $ne: 23 } }` |
| `$in`, `$nin` | In / Not in array | `{ role: { $in: ["A","B"] } }` |
| `$or` | Logical OR | `{ $or: [{...}, {...}] }` |
| `$and` | Logical AND (explicit) | `{ $and: [{...}, {...}] }` |
| `.count()` | Count matching docs | `db.col.find({}).count()` |
| `.forEach(fn)` | Iterate over results | `db.col.find().forEach(fn)` |
| Projection `{ field: 1 }` | Include field | `db.col.find({}, { name: 1 })` |
| Projection `{ field: 0 }` | Exclude field | `db.col.find({}, { age: 0 })` |
| `.sort({ field: 1 })` | Sort ascending | `db.col.find().sort({ age: 1 })` |
| `.sort({ field: -1 })` | Sort descending | `db.col.find().sort({ age: -1 })` |
| `.limit(n)` | Limit results | `db.col.find().limit(5)` |
| `.skip(n)` | Skip results | `db.col.find().skip(10)` |
| `"nested.field"` | Query nested docs | `db.col.find({ "a.b": true })` |
