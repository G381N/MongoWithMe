## 📦 What Are Collections & Documents?

Here's how MongoDB organizes data:

```
MongoDB Server
  └── Database (e.g., GebinDB)
        └── Collection (e.g., mydata)
              ├── Document { name: "Gebin", age: 23, ... }
              ├── Document { name: "Jerin", age: 23, ... }
              └── Document { name: "George", age: 25, ... }
```

If you're coming from SQL, here's the translation:

| SQL Term | MongoDB Term |
|:---------|:-------------|
| Database | Database |
| Table | Collection |
| Row | Document |
| Column | Field |

> **💡 Key Difference:** In MongoDB, documents in the same collection **don't need to have the same structure**. One document can have 3 fields, another can have 10 — that's totally fine!

## ✨ Creating a Collection

In MongoDB, you **don't** need to create a collection before using it. Just insert a document and MongoDB creates the collection automatically.

Make sure you're in the right database first:

```javascript
use GebinDB
```

```
switched to db GebinDB
```

## 🚀 Inserting Your First Document

Let's add our first document to a collection called `mydata`:

```javascript
db.mydata.insertOne({
  name: "Gebin",
  age: 23,
  phone: "9876543210",
  role: "FullStackDev"
})
```

**Output:**
```
{
  acknowledged: true,
  insertedId: ObjectId('699d49a83321373e437c2907')
}
```

That's it! MongoDB just:
1. Created a collection called `mydata` (automatically)
2. Inserted our document into it
3. Assigned it a unique `_id`

## 👥 Inserting Another Document

Let's add one more person:

```javascript
db.mydata.insertOne({
  name: "Jerin",
  age: 23,
  phone: "9876543210",
  role: "FullStackDev"
})
```

**Output:**
```
{
  acknowledged: true,
  insertedId: ObjectId('699d4aaf3321373e437c2908')
}
```

> Each document gets a different `ObjectId` — MongoDB's way of giving every document a unique identifier.

## 📋 Quick Reference

| Command | What It Does |
|:--------|:-------------|
| `db.<collection>.insertOne({...})` | Insert a single document |
| `db.createCollection("name")` | Explicitly create a collection (optional) |
