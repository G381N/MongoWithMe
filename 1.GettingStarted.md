## 🤔 What is `mongosh`?

`mongosh` is the **MongoDB Shell** — a command-line interface that lets you interact with your MongoDB databases directly. Think of it as a playground where you can:

- Create and switch between databases
- Insert, read, update, and delete data
- Run administrative commands
- Test queries before writing application code

It comes bundled with MongoDB, so if you've installed MongoDB, you already have it!

---

## 🔌 Connecting to MongoDB

Open your terminal and simply run:

```bash
mongosh
```

By default, this connects to your local MongoDB instance at:

```
mongodb://localhost:27017
```

You should see something like this:

```
Current Mongosh Log ID:	65a1b2c3d4e5f6a7b8c9d0e1
Connecting to:		mongodb://127.0.0.1:27017
Using MongoDB:		7.x.x
Using Mongosh:		2.x.x

test>
```

> **💡 Note:** The `test>` prompt means you're connected to the default `test` database. Don't worry — we'll switch to our own database next!

### Connecting to a Specific Host

If your MongoDB is running on a different host or port, pass the full connection string:

```bash
mongosh "mongodb://localhost:27017"
```

### Connecting to MongoDB Atlas (Cloud)

If you're using MongoDB Atlas, your connection string will look like:

```bash
mongosh "mongodb+srv://username:password@cluster0.xxxxx.mongodb.net/myDatabase"
```

## 🗂️ Navigating Databases

Once you're inside the shell, these are the commands you'll use all the time to move around.

### Show All Databases

```javascript
show dbs
```

**Output:**
```
admin    40.00 KiB
config  108.00 KiB
local    40.00 KiB
```

> This lists every database on your server along with its size. The three databases above (`admin`, `config`, `local`) are MongoDB's internal system databases.

### Switch to (or Create) a Database

```javascript
use GebinDB
```

**Output:**
```
switched to db GebinDB
```

> **🧠 Did you know?** If the database doesn't exist yet, MongoDB will create it automatically when you first insert data into it. Until then, it won't show up in `show dbs`. So don't panic if your new database seems to "disappear"!

### Show Collections in the Current Database

```javascript
show collections
```

**Output:**
```
mydata
users
products
```

> **Collections** are like "tables" in SQL databases — they hold groups of documents. If your database is new and empty, this command will return nothing.

### Check Which Database You're Currently In

```javascript
db
```

**Output:**
```
GebinDB
```

> This is handy when you have multiple terminals open and need to confirm which database you're working with.

## 📋 Quick Reference

| Command | What It Does |
|:--------|:-------------|
| `mongosh` | Open the MongoDB Shell (connects to `localhost:27017`) |
| `mongosh "connection_string"` | Connect to a specific MongoDB instance |
| `show dbs` | List all databases on the server |
| `use <dbName>` | Switch to a database (creates it if it doesn't exist) |
| `show collections` | List all collections in the current database |
| `db` | Display the name of the current database |
| `exit` | Exit the MongoDB Shell |
| `cls` | Clear the shell screen |
