## 📚 Retrieve All Documents — `find()`

The `find()` method retrieves **all documents** in a collection. Call it with no arguments to get everything:

```javascript
db.mydata.find()
```

**Output:**
```
[
  {
    _id: ObjectId('699d49a83321373e437c2907'),
    name: 'Gebin',
    age: 23,
    phone: '9876543210',
    role: 'FullStackDev'
  },
  {
    _id: ObjectId('699d4aaf3321373e437c2908'),
    name: 'Jerin',
    age: 23,
    phone: '9876543210',
    role: 'FullStackDev'
  }
]
```

> **💡 What's happening here?** `find()` returns a **cursor** — think of it as a pointer that walks through all the matching documents. The shell automatically iterates through the cursor and prints the results for you.

### What If There Are Too Many Documents?

When your collection has more than 20 documents, `mongosh` will display the first 20 and then show:

```
Type "it" for more
```

Just type `it` and press Enter to see the next batch!

## 🎯 Retrieve a Single Document — `findOne()`

When you only need **one** document, use `findOne()`. It returns the first document that matches your query:

```javascript
db.mydata.findOne({ name: "Gebin" })
```

**Output:**
```
{
  _id: ObjectId('699d49a83321373e437c2907'),
  name: 'Gebin',
  age: 23,
  phone: '9876543210',
  role: 'FullStackDev'
}
```

> **Notice:** `findOne()` returns the document **directly** (not wrapped in an array). If no document matches, it returns `null`.

You can also call `findOne()` with no arguments to get the first document in the collection:

```javascript
db.mydata.findOne()
```

---

## ⚖️ `find()` vs `findOne()` — When to Use Which

| | `find()` | `findOne()` |
|:---|:---------|:------------|
| **Returns** | A cursor (iterable list) | A single document |
| **Result format** | Array of documents `[{...}, {...}]` | One document `{...}` or `null` |
| **Matches** | All matching documents | Only the first match |
| **Use when** | You need multiple results | You need just one result |
| **Example** | Listing all users | Looking up a user by name |

> **🧠 Rule of thumb:** If you're looking for a specific document (like looking up by ID or name), use `findOne()`. If you want to browse or search through many documents, use `find()`.

---

## 📋 Quick Reference

| Command | What It Does |
|:--------|:-------------|
| `db.<collection>.find()` | Retrieve all documents in a collection |
| `db.<collection>.find({filter})` | Retrieve documents matching a filter |
| `db.<collection>.findOne({filter})` | Retrieve the first matching document |
| `db.<collection>.findOne()` | Retrieve the first document (any) |
| `it` | Show the next batch of results (when paginated) |
