## 🔧 The `$set` Operator

`$set` is the most important update operator. It lets you **add new fields** or **change existing ones** without touching the rest of the document.

```javascript
// ✅ Correct — updates just the age field
{ $set: { age: 24 } }

// ❌ Wrong — would REPLACE the entire document with just { age: 24 }
{ age: 24 }
```

> Always use `$set` for targeted updates. Without it, MongoDB replaces the whole document!

---

## 📄 Updating a Single Document — `updateOne()`

`updateOne()` takes two arguments — a **filter** (which document) and an **update** (what to change).

Let's add social media info to Gebin's document:

```javascript
db.mydata.updateOne(
  { name: "Gebin" },
  { $set: { socials: { hasGithub: true, hasLinkedin: true } } }
)
```

**Output:**
```
{
  acknowledged: true,
  insertedId: null,
  matchedCount: 1,
  modifiedCount: 1,
  upsertedCount: 0
}
```

Let's verify it worked:

```javascript
db.mydata.findOne({ name: "Gebin" })
```

**Output:**
```
{
  _id: ObjectId('699d49a83321373e437c2907'),
  name: 'Gebin',
  age: 23,
  phone: '9876543210',
  role: 'FullStackDev',
  socials: { hasGithub: true, hasLinkedin: true }
}
```

> Notice how `socials` is a **nested document** — an object inside our document. MongoDB handles this natively.

---

## 📚 Updating Multiple Documents — `updateMany()`

Want to update **every** document in the collection? Use `updateMany()` with an empty filter `{}`:

```javascript
db.mydata.updateMany(
  {},
  { $set: { socials: { hasGithub: true, hasLinkedin: true } } }
)
```

> **⚠️ Careful:** An empty filter `{}` matches **all documents**. Always double-check before running `updateMany()`!

---

## 📝 Adding Arrays to Documents

MongoDB handles arrays natively. Let's give everyone an `education` field:

```javascript
db.mydata.updateMany(
  {},
  { $set: { education: ["BCA", "MCA"] } }
)
```

Now every document has an `education` array:

```javascript
db.mydata.findOne({ name: "Gebin" })
```

**Output:**
```
{
  _id: ObjectId('699d49a83321373e437c2907'),
  name: 'Gebin',
  age: 23,
  phone: '9876543210',
  role: 'FullStackDev',
  socials: { hasGithub: true, hasLinkedin: true },
  education: [ 'BCA', 'MCA' ]
}
```

> **💡 Fun fact:** In SQL, storing a list of values requires a separate table and a JOIN. In MongoDB, arrays are just another field type!

---

## 📋 Quick Reference

| Command | What It Does |
|:--------|:-------------|
| `db.<col>.updateOne({filter}, {$set: {...}})` | Update the first matching document |
| `db.<col>.updateMany({filter}, {$set: {...}})` | Update all matching documents |
| `db.<col>.updateMany({}, {$set: {...}})` | Update ALL documents (empty filter) |
| `$set` | Add new fields or change existing ones |
